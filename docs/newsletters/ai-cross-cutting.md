## 3. Agentic-AI Deep Dive (5+ sources)

The AE specifically needs to be fluent on tool-use frameworks, MCP, browser agents, voice agents, and agent infra — because every "AI for legal" pitch is now an agent pitch.

1. **Latent Space** — `latent.space` — The AI Engineer publication. Weekly essays + podcast cover agent harnesses, MCP adoption, eval design, tool-use. swyx is the closest thing the agent ecosystem has to an editor-in-chief. **Start here.**

2. **Smol AI / AINews** — `news.smol.ai` — Daily auto-aggregated chatter from every major agent-builder Discord (LangChain, LlamaIndex, Anthropic, OpenAI, Cursor, etc.). Best early-signal source for new MCP servers, new agent frameworks, new browser-agent demos. Karpathy-endorsed.

3. **Cobus Greyling** — `cobusgreyling.substack.com` + `cobusgreyling.medium.com` — The most prolific agent-architecture explainer. Posts 2-3x/week on multi-agent systems, harnesses, agent reliability. Great for the AE who needs to draw the architecture diagram on the whiteboard.

4. **LangChain Blog — In the Loop series** — `blog.langchain.com/tag/in-the-loop/` — Harrison Chase's essays on what's hard about production agents (human-in-the-loop, interrupts, durable execution). Vendor blog, but LangChain sets framework norms.

5. **Chain of Thought (Every)** — `every.to/chain-of-thought` — Dan Shipper's PM-operator lens: he ships agent products inside Every and writes about the seams. Pairs well with Latent Space's engineer lens.

6. **Interconnects** — `interconnects.ai` — Nathan Lambert covers the *model* side that determines what agents can actually do (post-training, RLHF, reasoning models). The "why is this agent now possible" answer usually lives here.

7. **Inference by Sequoia** — `inferencebysequoia.substack.com` — Sequoia's AI thesis newsletter; agent-economy framing for the business side of the AE's brain.

## Notes on Implementation for Dugout

- **Sender-domain coverage:** the stack mixes Substack (`*.substack.com` and custom domains via Substack), Ghost (`platformer.news`), Beehiiv (`aidailybrief.beehiiv.com`), Mailchimp (`stanford.us18.list-manage.com`), Buttondown legacy (`news@smol.ai`), and corporate (`deeplearning.ai`, `mckinsey.com`, `bcg.com`, `a16z.com`, `cset.georgetown.edu`, `bain.com`, `theinformation.com`). Classifier should handle each.
- **Forwarding posture:** Substack and Ghost newsletters forward cleanly. The Information uses account-bound paid emails — flag for the AgentMail webhook (sender-on-paid-substrate ≠ shareable internally).
- **Paid sources flagged in table:** The Information, Platformer paid tier, Interconnects paid tier, Ahead of AI paid tier, AI Tidbits paid tier, Big Technology paid tier, Ben's Bites community. Free tier of each is sufficient for ingestion in most cases except The Information.
- **Plain-HTML body:** all 35 sources verified as plain-HTML email (no required JS render). Latent Space, Interconnects, Ahead of AI, and Don't Worry About the Vase frequently exceed Gmail's 102KB clip threshold — parser should handle "[Message clipped] View entire message" gracefully.
