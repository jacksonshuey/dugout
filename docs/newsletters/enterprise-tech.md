## Notes on Sender Domains (for AgentMail webhook routing)

- **Substack** newsletters all send from `<slug>@substack.com` by default unless the publisher set up a custom domain. Most of the entries above use the default. Add a single rule for `@substack.com` and tag based on the From display name.
- **Beehiiv** (Ben's Bites, some others) sends from either the publisher's own domain or `mail.beehiiv.com` -- check actual headers after first email.
- **Kit / ConvertKit** (Tunguz) typically uses a verified sender on the author's own domain (e.g. `tom@tomtunguz.com`).
- **Ghost** (Runtime, Platformer) sends from the publisher's own domain.
- **Industry Dive properties** (CIO Dive) send from `@industrydive.com`.
- **Axios** sends from `@axios.com` (or `link.axios.com` for transactional).
- **Fortune** sends from `@fortune.com`.
- **The Information** sends from `@theinformation.com`.

For the classifier, the highest-value `from_domain` allowlist (top 9):
1. `runtime.news`
2. `theinformation.com`
3. `cloudwars.com`
4. `industrydive.com` (CIO Dive)
5. `axios.com`
6. `fortune.com`
7. `dataengineeringweekly.com` / `substack.com` (filter on display name)
8. `snowflake.com` + `databricks.com` (vendor official)
9. `platformer.news`

Substack entries should be matched by `from_name` (e.g. "Jamin Ball", "Gergely Orosz", "Tristan Handy", "Benn Stancil", "Jack Clark") since they all share `substack.com`.
